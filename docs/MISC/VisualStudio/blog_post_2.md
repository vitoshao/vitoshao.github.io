這篇文章要介紹，在 Visual Studio 中的「hot-key」功能

# 工具視窗

| Hot Key | 說明 |
| --- | --- |
| Ctrl + Tab | 在已開啟的各視窗之間快速進行切換 (還可以順便看到各視窗的縮圖)。 |
| Ctrl + Alt + O | 開啟輸出視窗 |
| Ctrl + Alt + X | 開啟工具箱 |
| F4 | 開啟屬性視窗 |
| F7   &lt;br&gt; Shift + F7 | View Code   &lt;br&gt; View Designer |

# 游標位置

| Hot Key | 說明 |
| --- | --- |
| F12 | 移至定義 (Go To Definition) |
| Ctrl + 減號  &lt;br&gt;Ctrl + Shift + 減號 | 上一步的位置 (回到移至定義前的游標位置)  &lt;br&gt;下一步的位置 |
| Ctrl + PgUp  &lt;br&gt;Ctrl + PgDn | 快速的將游標移動到當頁的最上方  &lt;br&gt;快速的將游標移動到當頁的最下方 |
| F3   &lt;br&gt; Shift + F3 | 尋找下一筆   &lt;br&gt;尋找上一筆 |
| F9   &lt;br&gt; Ctrl + Shift + F9 | 插入中斷點  &lt;br&gt;清除所有中斷點 |
| Ctrl + K + K  &lt;br&gt; Ctrl + K + N  &lt;br&gt;Ctrl + K + P | 設定書籤  &lt;br&gt;移到下一個書籤  &lt;br&gt;移到上一個書籤 |

# 選取、編輯、排版

| Hot Key | 說明 |
| --- | --- |
| Alt + [滑鼠左鍵] | 選取四方型的區塊 |
| Shift + HOME  &lt;br&gt;Shift + END | 選取游標位置前的文字  &lt;br&gt;選取游標位置後的文字 |
| Ctrl + L | 刪除游標所在的那一行 |
| Ctrl + K + C  &lt;br&gt;Ctrl + K + U | 選取範圍,加上註解  &lt;br&gt;選取範圍,取消加上註解 |
| Ctrl + K + F  &lt;br&gt;Ctrl + K + D | 將選取範圍的程式碼自動排版  &lt;br&gt;將整份文件的程式碼自動排版 |
| Ctrl + ] | 檢查匹配括號(在左右括號間切换) |
| Ctrl + M + P  &lt;br&gt;Ctrl + M + O  &lt;br&gt;Ctrl + M + M | 打開所有的 Region 區塊  &lt;br&gt;合上所有的 Region 區塊  &lt;br&gt;關閉或打開游標所在的 Region 區塊 |
| Shift + Alt + Enter | 將程式碼視窗變成全螢幕 |

# 編譯、建置

| Hot Key | 說明 |
| --- | --- |
| F6 或 Ctrl + Shift + B | 建置方案 |
| Alt + B + R | 重建方案 |
| Ctrl + Break | 取消建置 |
| F5 | Build & Run |
| F6 | Build |
| F10 | Step Over |
| Ctrl + F10 | 執行到游標所在列 |
| F11 | Step Into (單步執行 & 進入副程式) |
| Shift + F11 | Step Out |

#### PS:「建置方案」與「重建方案」有什麼不同？

- 建置專案會事先檢查原始碼，如果有修改，則重新編譯，沒有修改的部份，則不需要重新編譯。
- 重建專案則不管原始碼是否有修改過，方案下所有的程式碼，都會重新編譯。
